1. Hosting & Deployment

▲ Vercel

⭐ Our Pick

The gold standard for Next.js and frontend deployment. Edge functions, instant rollbacks, and analytics built in. We deploy every ZOO frontend project here.

✅ Pros

  • • Zero-config Next.js deployment
  • • Edge functions with 100+ locations
  • • Built-in analytics & speed insights
  • • Preview deployments for every PR
  • • Generous free tier (100GB bandwidth)

❌ Cons

  • • Can get expensive at scale
  • • Serverless function time limits
  • • Vendor lock-in with Next.js
Pricing: Free → $20/mo (Pro) → $0.18/GB bandwidth Try Vercel Free →

🚂 Railway

Best for Full-Stack

Deploy databases, APIs, and workers in one platform. Usage-based pricing means you only pay for what you use. Perfect for AI agents that need background workers.

✅ Pros

  • • Deploy anything (Docker, Node, Python)
  • • Built-in PostgreSQL, Redis, MongoDB
  • • Usage-based pricing (pay per CPU/RAM second)
  • • Great for background workers & cron jobs
  • • $5 free credit monthly

❌ Cons

  • • Less mature than Vercel for frontend
  • • Can be confusing to estimate costs
  • • Smaller community
Pricing: $5/mo free credit → usage-based from $0.000231/min Try Railway →

  • • You manage your own scaling

2. Database & Backend

⚡ Supabase

⭐ Our Pick

Open-source Firebase alternative with PostgreSQL. Real-time subscriptions, auth, storage, and edge functions. Every ZOO AI agent uses Supabase for persistent memory.

✅ Pros

  • • Full PostgreSQL (not NoSQL)
  • • Real-time subscriptions built-in
  • • Auth with 20+ providers
  • • Storage with CDN
  • • Self-hostable (open source)
  • • Generous free tier (500MB DB)

❌ Cons

  • • Can be slow on free tier
  • • Complex queries need SQL knowledge
  • • Edge functions still maturing
Pricing: Free (500MB) → $25/mo (Pro) → $0.00325/GB Try Supabase Free →

🔥 Firebase

Best for Mobile

Google's BaaS platform. Firestore, Auth, Cloud Functions, and ML Kit. Best choice if you're building mobile apps with Flutter.

✅ Pros

  • • Best-in-class mobile SDKs
  • • Firestore is fast and scalable
  • • Google Cloud integration
  • • ML Kit for on-device AI

❌ Cons

  • • NoSQL only (no PostgreSQL)
  • • Vendor lock-in
  • • Can get very expensive
  • • Not open source
Pricing: Free (Spark) → Pay-as-you-go (Blaze) Try Firebase →
